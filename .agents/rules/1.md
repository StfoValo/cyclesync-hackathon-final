## The Stack:

## Backend: FastAPI (Python).
 It natively supports async, which we need for our LLM streaming.

## Frontend:
 Vanilla HTML/JS with Tailwind CSS (via CDN) to rapidly recreate the dark-theme UI. We will use Chart.js for data visualization.

## Strict Refactoring Rules:

1.**Preserve the Models:** The models/ and mcp_agent_server/ directories contain complex actuarial math, physics simulations, and LLM prompts. Treat these as sacred. Do not change the core logic; only adapt import paths if necessary.

2.**Controllers become Routers:** The PyQt6 controllers/ will be transformed into FastAPI routers (e.g., @app.get("/api/fleet/kpis")). They must return JSON payloads.

3.**Views become Frontend Components:** The PyQt6 views/ will be rewritten as modular HTML/JS files.

4.**No PyQt6:** Completely strip out all PyQt6 imports (QWidget, pyqtSignal, QThread, QWebEngineView).

5.**Streaming AI:** Convert the QThread AI streaming logic into Server-Sent Events (SSE) using FastAPI's StreamingResponse.

6.**Documentations:** Always document what you do inside the README.md (so you will have to create the necessary files to provide the right documentation for the refactoring you are going to do)
